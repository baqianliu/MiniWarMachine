## 最终目标
* 主要是人形但不限于人形的模块化机器人，方便的拆卸和组装
* 支持实时控制，这是着重需要实现的的功能
* 完成实时控制软件，上架Steam，可以实现本地和线上两种模式的实时对战
* 尺寸大小没有限制，先做缩略版，再做真人大小版
* 支持自动运行和多种方式的手动控制，主要手动控制方式为AR或者键鼠方式，实时图像+现实增强+体感控制
* 支持机器人动作的录入与回放，一键便可以精确执行录入过的动作
* 有现场控制版本(不通过屏幕控制)，可以加装头部远程控制模块升级为可远程控制版本
* 包含两个控制核心，头部为主要控制核心，胸部为副控制核心
* 支持集群控制，一脑多从
* 使用ChatGPT进行自编程

## 关键问题解决思路
### 网路延迟的降低
* 5G通信技术：5G网络是目前最先进且最适合低延迟应用的通信技术之一。5G网络的设计目标之一就是实现低至1毫秒的端到端延迟。以下是一些具体的措施：
* 边缘计算：将计算资源部署在网络的边缘，靠近数据源和用户，减少数据传输距离，从而降低延迟。
* 网络切片：利用5G网络切片技术，为特定的应用（如远程医疗）分配专用的网络资源，确保高优先级的数据传输。
* QoS（服务质量）保障：配置高质量的服务保障，确保关键数据的优先传输。
* 短路径路由：选择最短的传输路径，减少数据包在网络中的跳数。
* 专网建设：在医院和操作中心之间建立专用网络，减少公共网络的不确定性和干扰。
* 高性能硬件：使用高性能的服务器、路由器和交换机，确保数据处理和传输的速度。
* 协议优化：使用高效的网络协议，如QUIC（Quick UDP Internet Connections），减少握手时间和重传延迟。
* 缓存技术：在关键节点上使用缓存技术，减少数据的重复传输。
* 网络监控：实时监控网络状态，及时发现和解决潜在的问题。
* 冗余备份：设置冗余的网络路径和备份系统，确保在网络故障时能够快速切换。
* 云计算：利用云计算资源进行数据处理和存储，减少本地计算的压力。
* 边缘计算：将部分计算任务下放到边缘设备，减少数据传输的距离和时间。
* 网络服务提供商：选择有经验的网络服务提供商，他们通常有专业的团队和技术来确保低延迟的网络连接。
* 数据预处理：对触觉反馈和控制信号进行预处理，减少不必要的数据传输。
